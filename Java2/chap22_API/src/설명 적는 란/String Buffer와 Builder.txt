StringBuffer 와 StringBuilder
	공통점 :
		둘 다 문자열을 다루는 클래스
		문자열을 유동적으로 변경하는데 사용
		
StringBuffer
	가변클래스 
	스레드 안전(thread - safe) 
	멀티스레드 환경에서 여러 스레드가 동시에 접근해도 데이터 무결성 유지(안전함)
	동기화가 적용되어 성능 면에서는 Builder 보다 느림, or 느릴 수 있음
	
StringBuilder
	가변클래스(바뀌거나 바귈 수 있는 클래스)
	단일 스레드(하나의 스레드) 사용하기에 적합 / 성능이 빠름
	문자열을 더하거나 수정할 때 메모리를 절약할 수 있다
	동기화를 고려하지 않음
	스레드 간에 동기화 관련 문제가 없음
	스레드 간에 오버헤드
	
오버헤드(OverHead) 어떤 작업을 수행 할 때 추가적으로 필요한 자원, 시간, 비용 등
